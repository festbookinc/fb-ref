# 모바일 최적화 작업 목록 (FB Ref.)

> 전반적인 모바일 UX 개선을 위한 작업 목록입니다. 검토 후 진행 여부를 결정해 주세요.

---

## 1. 네비게이션 바 (Navbar)

**현상**
- 로고, 검색창, 태그, 추가, 다크모드, 마이페이지/로그인이 한 줄에 배치됨
- 모바일(320~430px)에서 요소가 넘치거나 겹침

**작업**
- 모바일에서 햄버거 메뉴 + 슬라이드 아웃 메뉴로 전환
- 또는 검색창을 아이콘으로 접고, 탭 시 전체 화면 검색 모달 표시

---

## 2. 이미지 상세 모달 (ImageDetailModal)

**현상**
- 이미지(좌) + 정보 패널(우, `w-96`) 가로 배치
- 모바일에서 가로 스크롤 발생 또는 레이아웃 깨짐

**작업**
- 모바일에서 세로 스택: 이미지 위, 제목·버튼·설명·댓글 아래
- 모달 전체 `overflow-y-auto`로 스크롤 가능하게

---

## 3. 이미지 그리드 (ImageArchiveGrid)

**현상**
- 카드 기본 280px → 좁은 화면에서 1열만 보이거나 카드가 과도하게 큼

**작업**
- 모바일 기본 카드 크기 축소 (예: 160px) 또는 미디어 쿼리로 `minmax` 조정
- 정렬/카드 크기 컨트롤을 모바일에서 더 컴팩트하게 배치

---

## 4. 모달 공통 (UploadModal, TagModal, LoginModal 등)

**현상**
- 모바일에서 패딩·높이 부족, 스크롤 불편

**작업**
- 모바일에서 `max-h-[90vh]`, `overflow-y-auto` 적용
- 패딩 `p-4` → `p-4 sm:p-6` 등 반응형 조정
- 버튼·입력 필드 터치 영역 확대

---

## 5. 터치 타겟 크기

**현상**
- 닫기, 다크모드 토글 등 작은 버튼이 터치하기 어려움

**작업**
- 주요 버튼 최소 44×44px (`min-h-[44px] min-w-[44px]`) 적용
- 패딩 확대로 클릭 영역 확대

---

## 6. Viewport & Safe Area

**현상**
- 노치·홈 인디케이터 영역 미대응 가능성

**작업**
- `layout.tsx`에 `viewport` 메타데이터 설정
- `env(safe-area-inset-*)`로 하단 네비게이션 등 여백 적용

---

## 7. 마이페이지

**현상**
- 보드 생성 폼이 모바일에서 좁음

**작업**
- 모바일에서 입력창·버튼 세로 배치 (`flex-col`)
- 보드 카드·이미지 그리드 모바일 레이아웃 점검

---

## 8. 입력 필드 (폼)

**현상**
- iOS에서 16px 미만 폰트 시 자동 줌 발생

**작업**
- `input`, `textarea`에 `text-base`(16px) 이상 적용
- `font-size: 16px` 확실히 지정

---

## 9. 가로 스크롤 방지

**작업**
- `body` 또는 `main`에 `overflow-x-hidden` 적용
- 고정 너비 요소가 뷰포트를 넘지 않도록 확인

---

## 10. 드래그 앤 드롭 (DropZone)

**현상**
- 모바일에서는 드래그가 거의 사용되지 않음

**작업**
- 모바일에서는 DropZone 비활성화 또는 "탭해서 업로드" 안내 추가 (선택)

---

## 우선순위 제안

| 우선순위 | 작업 | 예상 영향 |
|:---:|------|----------|
| 높음 | 1. Navbar 모바일 메뉴 | 전체 사용성 |
| 높음 | 2. ImageDetailModal 세로 레이아웃 | 상세 보기 사용성 |
| 중간 | 3. 이미지 그리드 카드 크기 | 목록 보기 |
| 중간 | 4. 모달 스크롤·패딩 | 업로드·태그 등 모달 사용성 |
| 중간 | 5. 터치 타겟 | 버튼 조작 편의성 |
| 낮음 | 6~10 | 세부 개선 |
