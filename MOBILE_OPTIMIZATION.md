# 모바일 최적화 전략 (FB Ref.) v2

> UI·기능 변경 반영 (2025). 검색 초기화 삭제, 좋아요, 마이페이지 탭, 이미지 카드 개선 등 적용 후 재정리.

---

## 현재 UI 구조 요약

| 영역 | 구성 | 모바일 이슈 |
|------|------|-------------|
| **Navbar** | 로고 + 검색창 + Tag 버튼 + (+) + 다크모드 + 마이페이지/로그인 | 320~430px에서 요소 과다, 한 줄 배치 |
| **ImageDetailModal** | 이미지(좌) + 정보 패널(우, w-96) 가로 배치 | 가로 스크롤, 패널 잘림 |
| **이미지 그리드** | ImageArchiveGrid, MyImagesGrid, MyLikesGrid | 카드 기본 280/220px, 컨트롤 바 |
| **마이페이지** | 저장한 이미지 / 보드 / 좋아요 탭 | 보드 생성 폼, 탭 버튼 |
| **모달** | UploadModal, TagModal, LoginModal, BoardAddModal | 패딩·스크롤·터치 영역 |

---

## 1. Navbar (네비게이션 바)

**현상**
- 로고, 검색창, Tag, +, 다크모드, 마이페이지가 한 줄에 배치
- 375px 이하에서 요소가 겹치거나 넘침

**전략 A: 햄버거 메뉴**
- 모바일(<640px)에서 햄버거 아이콘만 표시
- 탭 시 슬라이드 아웃 메뉴: 검색, Tag, +, 다크모드, 마이페이지
- 검색은 메뉴 내 인풋 또는 별도 검색 모달

**전략 B: 검색 아이콘화**
- 모바일에서 검색창을 돋보기 아이콘으로 대체
- 탭 시 전체 화면 검색 모달 (입력 + Tag 버튼 포함)
- 로고, Tag(아이콘), +, 마이페이지만 상단에 유지

**권장**: B (검색 모달) — 검색 UX가 모바일에서 더 자연스러움

---

## 2. ImageDetailModal (이미지 상세)

**현상**
- `flex` 가로 배치: 이미지 영역 + `w-96` 정보 패널
- 모바일에서 가로 스크롤 또는 패널 잘림

**전략**
- `md:` 브레이크포인트로 분기
  - **모바일**: `flex-col` — 이미지 위, 제목·설명·태그·댓글·액션 버튼 아래
  - **데스크톱**: 기존 가로 배치 유지
- 모달 전체 `overflow-y-auto`, `max-h-[90vh]` 유지
- 하단 액션 바(다운로드, 좋아요, 보드 추가, 편집/삭제) 터치 영역 확대

---

## 3. 이미지 그리드 (ImageArchiveGrid, MyImagesGrid, MyLikesGrid)

**현상**
- 기본 카드 크기 280px(메인) / 220px(마이페이지)
- `minmax(cardSize, 1fr)` → 좁은 화면에서 1열만 보이거나 카드가 과도하게 큼
- 정렬·카드 크기 컨트롤이 모바일에서 공간 차지

**전략**
- 모바일 기본 카드 크기: `min(100%, 160px)` 또는 `clamp(140px, 45vw, 280px)`
- 컨트롤 바: 모바일에서 `flex-col` 또는 드롭다운으로 접기
- MyImagesGrid 선택 모드: 모바일에서 버튼 텍스트 짧게 ("선택 해제" → "해제" 등) 또는 아이콘화

---

## 4. 마이페이지

**현상**
- 탭: 저장한 이미지 | 보드 | 좋아요
- 보드 생성 폼: 입력 + 버튼 가로 배치
- 보드 카드: 썸네일 그리드, 수정/삭제 버튼

**전략**
- 탭 버튼: 모바일에서 `text-sm` 또는 아이콘+라벨 축약
- 보드 생성 폼: 모바일 `flex-col`, 버튼 전체 너비
- 보드 카드: 모바일 1열, 터치 영역 확대

---

## 5. 모달 공통 (UploadModal, TagModal, LoginModal, BoardAddModal)

**현상**
- 패딩·높이 부족, 스크롤 불편
- 입력 필드 터치 영역 작음

**전략**
- `max-h-[90vh] overflow-y-auto` 적용
- 패딩: `p-4 sm:p-6`
- 버튼·입력: `min-h-[44px]` (Apple HIG 권장)
- input/textarea: `text-base`(16px) 이상 → iOS 자동 줌 방지

---

## 6. 터치 타겟

**현상**
- 닫기, 다크모드, 좋아요, 다운로드 등 `p-2` 버튼이 44×44px 미만

**전략**
- 주요 버튼: `min-h-[44px] min-w-[44px]` 또는 `p-3`
- ImageDetailModal 하단 액션: `p-3` 적용

---

## 7. Viewport & Safe Area

**전략**
- `layout.tsx`: `viewport` 메타데이터 설정
- 하단 고정 요소: `pb-[env(safe-area-inset-bottom)]` 적용
- `overflow-x-hidden` on body/main으로 가로 스크롤 방지

---

## 8. 드래그 앤 드롭 (DropZone)

**현상**
- 모바일에서 드래그 거의 미사용

**전략**
- 모바일: "탭해서 업로드" 안내 문구 추가
- 또는 DropZone 영역 탭 시 파일 선택 다이얼로그 트리거

---

## 9. 보드 상세 (BoardDetailContent)

**현상**
- 보드별 이미지 그리드, 모바일 레이아웃

**전략**
- 그리드 `grid-cols-2` 모바일 유지, 카드 크기 조절
- 이미지 추가/삭제 버튼 터치 영역 확대
