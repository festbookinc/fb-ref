## 📋 프로젝트 개요

| 항목 | 내용 |

## 📊 Supabase DB 스키마 (예정)

```
users (NextAuth 연동 또는 Supabase Auth)
├── id, email, name, image, created_at

images
├── id, title, description, link, image_url, user_id
├── created_at, updated_at

tags
├── id, name

image_tags (다대다)
├── image_id, tag_id

comments
├── id, image_id, user_id, content, created_at

boards
├── id, name, user_id, created_at

board_images (다대다)
├── board_id, image_id, created_at
```

## ⚠️ 계정 사용 규칙 (필수)

**GitHub, Supabase, Vercel 등 외부 서비스 사용 시, 반드시 사용자에게 어떤 계정을 사용할지 먼저 물어봅니다.**

- GitHub: 저장소 생성/푸시 전에 **어떤 GitHub 계정** 사용할지 확인
- Supabase: 프로젝트 생성/연결 전에 **어떤 Supabase 계정** 사용할지 확인
- Vercel: 배포 설정 전에 **어떤 Vercel 계정** 사용할지 확인

→ AI가 임의로 계정을 선택하지 않습니다. 항상 사용자 확인 후 진행합니다.
